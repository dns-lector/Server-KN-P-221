Моделі. Взаємодія з БД

Моделі (з процесу моделювання) - редуковані представлення реальних об'єктів із залишком лише необхідних для проєкту характеристик.
Модель (з MVC-like патернів) - частина архітектури проєкту, яка відповідає за взаємодію з постачальниками даних. Зазвичай, у вигляді об'єктів-моделей.
У Django моделі - класи, що описують дані, які передаються до БД.

  • Моделі оголошуються у файлі models.py, що є частиною базового проєкту
  • Після оголошення моделей необхідно створити міграцію (якщо сервер запущений, то його краще зупинити)
    python manage.py makemigrations app221
    Migrations for 'app221':
    app221\migrations\0001_initial.py
    + Create model Client
  • Виконуємо створену міграцію
    python manage.py migrate
    Running migrations:
    Applying app221.0001_initial... OK

Панель адміністратора також входить до базового проєкту і дозволяє взаємодіяти з БД.

  • Створюємо користувача-адміністратора (супер-користувача)
    python manage.py createsuperuser
    Superuser created successfully
  • Запускаємо сервер, якщо це не зроблено раніше, переходимо за адресою /admin
  • Моделі, навіть після міграції, не потрапляють автоматично до панелі адміністратора. Для включення їх необхідно зареєструвати у файлі admin.py (є частиною базового проєкту)
    admin.site.register(Client)
    Оновлюємо сайт-адмінку, створюємо через нього нового клієнта, переконуємось у його відображенні
  • Для покращення відображення створємо адмін-модель та реєструємо її співвідношення з моделлю даних у файлі admin.py
    class ClientAdmin(admin.ModelAdmin) :
      list_display = ('id', 'last_name', 'first_name', 'register_at')

    admin.site.register(Client, ClientAdmin)
